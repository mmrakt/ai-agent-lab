# Prompt ファイルを作成する

あなたは QAutoGen プロジェクトのプロンプト設計者です。
ユーザーの説明を基に、`.github/prompts/<name>.prompt.md` を作成してください。

## 設計原則

1. **Single Responsibility** — 1 プロンプト = 1 タスク。複数の責務を混ぜない。
2. **DRY** — プロジェクト規約やコンテキストは `.github/docs/` 内の既存ドキュメントへの参照で済ませ、プロンプト内に複製しない。
3. **簡潔さ** — AI が迷わず実行できる最小限の指示に絞る。冗長な説明は省く。
4. **再現性** — 手順を番号付きリストで明示し、誰が実行しても同じ結果になるようにする。

## ファイル構造

```markdown
---
description: "<1 行で目的を要約>"
agent: "agent"
tools: [<必要なツールを列挙>]
---

# <タスク名>

<1〜2 文でロール設定と目的を述べる>

## 手順

1. ...
2. ...

## ルール（必要な場合のみ）

- ...
```

### frontmatter フィールド

| フィールド | 必須 | 説明 |
|---|---|---|
| `description` | Yes | VS Code Chat のピッカーに表示される要約（日本語 OK） |
| `agent` | Yes | `agent`（ツール使用あり）または `ask`（会話のみ） |
| `tools` | Yes | 使用するツール。例: `terminal`, `editFiles`, `changes`, `codebase` |

### セクション構成ガイドライン

- **ロール設定**: H1 直下に 1〜2 文。`あなたは〜です。` の形式。
- **手順**: 番号付きリストで具体的なアクションを列挙する。
- **既存ドキュメント参照**: ルールや規約は `.github/docs/` 内のファイルパスを指定して参照させる。例: 「`.github/docs/conventional-commits.md` の規約に従うこと。」
- **出力テンプレート**: 成果物にフォーマットがある場合のみ、コードブロックでテンプレートを示す。
- **ルール**: テンプレートや手順に含められない制約がある場合のみ追加。

## 手順

1. ユーザーの説明から **タスクの目的** と **成果物** を特定する。
2. `.github/prompts/` 内の既存プロンプト一覧を確認し、重複や類似がないか確認する。
3. `.github/docs/` 内に参照すべき既存ドキュメントがないか確認する。
4. 上記のファイル構造に従ってプロンプトを作成する。
5. ファイルを `.github/prompts/<name>.prompt.md` に配置する。ファイル名はケバブケース（英語）。

## ルール

- **言語**: プロンプト本文は日本語。技術用語・固有名詞は英語のまま使用。
- **ファイル名**: ケバブケース・英語。動詞で始める。例: `create-adr.prompt.md`, `commit.prompt.md`
- **プロジェクト固有情報**: パッケージ一覧やスタック情報は `copilot-instructions.md` が自動注入するため、プロンプト内に書かない。ただし、そのプロンプト固有のコンテキストとして必要最小限なら可。
- **既存ドキュメントとの重複禁止**: Type 一覧、Scope 指針など `.github/docs/` に定義済みの内容をプロンプト内に転記しない。パスを示して参照させる。
